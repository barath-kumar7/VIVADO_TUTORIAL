Getting Started

This webpage is designed to help you start learning Verilog and setting up Vivado.

Verilog Installation Guide

Follow these steps to install Vivado and set up your Verilog environment:

  1. Open your preferred browser (Chrome or Edge) and navigate to the following link: Xilinx Vivado Download.
  2. If you do not have an account, you will need to create one. If you already have an account, simply log in.
  3. Once logged in, download the Vivado installer for your operating system (Windows/Linux).
  4. After the download is complete, open the file manager and run the installer setup file.
  5. During the installation, select the *ML Standard* edition. This edition includes all necessary tools for Verilog development.
  6. The installer will prompt you to install additional files (approximately 22GB). It is recommended that you do not untick any of the default selections to ensure that all required components are installed.
  7. Installation may take some time depending on your internet speed and system performance. Please be patient as the process completes.
  8. Once the installation is finished, you will be ready to start using Vivado for Verilog development.

Basics of Verilog

This section covers the fundamentals of Verilog, including syntax, data types, and modules.

Syntax

Verilog syntax follows standard conventions:

  • Comments: Single-line (//) and multi-line (/*...*/).

Data Types

  • Registers: Store values.
    • reg: Holds its value until changed.
    reg [7:0] data; // 8-bit register

Modules

Modules are the building blocks in Verilog. A typical module definition includes ports, internal variables, assignments, and always blocks.

  • Module Definition:
    module module_name (port_list); // Port Declarations // Internal Variables // Continuous Assignments // Always Blocks endmodule
  • Example - AND Gate:
    module and_gate (output wire y, input wire a, b); assign y = a & b; // Continuous assignment endmodule

Functions

For repetitive code, functions cannot contain timing controls:

function [3:0] func_name; input [3:0] a; // Function Body endfunction

Parameterization

Reusable and configurable modules using parameters:

module param_module #(parameter WIDTH = 8) (input [WIDTH-1:0] in, output [WIDTH-1:0] out); // Module Body endmodule

Generate Statements

Used for creating multiple instances of a module or block of code:

genvar i; generate for (i = 0; i < 8; i = i + 1) begin : loop_label // Module instances or statements end endgenerate

Initial Blocks

Used for initializing variables and running simulation-only code:

initial begin // Initialization statements end

Delays

Introduce delays in assignments:

assign #5 out = in; // 5 time units delay

Continuous Assignments

For combinational logic:

assign y = a & b;

Blocking vs. Non-blocking Assignments

Blocking (=) executes sequentially, Non-blocking (<=) executes in parallel.

a = b; // Blocking
a <= b; // Non-blocking
